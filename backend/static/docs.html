API Reference

Base URL: your deployment's origin, e.g. http://localhost:5000 locally or https://doc-scanner.thejaydenproject.com on the live demo.
All endpoints return JSON. All errors include an error string field.

Async Job Flow

POST /api/documents/scan → 202 { job_id }
    ↓ poll every 4s, up to 30 attempts (90 if stage becomes "upscaling")
GET /api/documents/jobs/<job_id> → { status: "processing" }
    ↓ when ready, or DELETE the job to cancel it early
GET /api/documents/jobs/<job_id> → { status: "complete", result: { ... } }
POST /api/documents/scan

Accepts an image upload, validates it, dispatches the CV pipeline as a Celery task, and immediately returns a job ID. Subject to rate limiting: 20 requests per IP per hour, 1 concurrent scan per IP, 3 concurrent scans globally.

Field Type Notes
file multipart/form-data JPEG or PNG, max 20MB
Status Body Condition
202 { job_id } Job accepted
400 { error } Missing file, wrong MIME type, empty file
429 { error } Rate limit exceeded or server at capacity 
// Request
POST /api/documents/scan
Content-Type: multipart/form-data
Body: file=<image.jpg>

// Response 202
{ "job_id": "a3f2c1d4-..." }
GET /api/documents/jobs/<job_id>

Polls the status of a scan job. Returns processing (optionally with stage: "upscaling"), complete, failed, or cancelled. The frontend polls every 4 seconds and normally gives up after 30 attempts (2 minutes); once it sees stage: "upscaling" that budget triples to 90 attempts (6 minutes), since FSRCNN super-resolution plus OCR routinely takes longer than the base budget on the CPU-only VPS. Giving up cancels the job and shows a timeout error.

Status Body Condition
200 { status, result? } Job found
404 { error } Job ID not found 
// Response — complete
{
  "status": "complete",
  "result": {
    "text": "extracted text here...",
    "char_count": 342,
    "word_count": 61,
    "processing_time_ms": 1240,
    "warped_image_b64": "iVBORw0KGgo...",
    "detection_count": 47,
    "doc_type": "handwritten",
    "doc_type_confidence": 0.7821,
    "doc_type_source": "model"
  }
}

// Response — failed
{
  "status": "failed",
  "error": "No four-point document contour found."
}

// Response — cancelled (via DELETE, see below)
{
  "status": "cancelled"
}
DELETE /api/documents/jobs/<job_id>

Cancels a scan. If it's still processing, the Celery task is forcefully terminated (SIGKILL). A no-op if the job already finished or doesn't exist; both still return 200.

Status Body Condition
200 { status: "ok" } Cancelled, or no-op (already finished / unknown ID) 
// Request
DELETE /api/documents/jobs/a3f2c1d4-...

// Response 200
{ "status": "ok" }

Returns aggregated statistics across all scans plus the 10 most recent records. Used to populate the Chart.js dashboard.
