API Reference

Base URL: https://your-tunnel-url.trycloudflare.com
All endpoints return JSON. All errors include an error string field.

Async Job Flow

POST /api/documents/scan → 202 { job_id }
    ↓ poll every 1.5s
GET /api/documents/jobs/<job_id> → { status: "processing" }
    ↓ when ready
GET /api/documents/jobs/<job_id> → { status: "complete", result: { ... } }
POST /api/documents/scan

Accepts an image upload, validates it, submits the CV pipeline to a background thread, and immediately returns a job ID. Subject to rate limiting: 20 requests per IP per hour, 1 concurrent scan per IP, 3 concurrent scans globally.

Field Type Notes
file multipart/form-data JPEG or PNG, max 2MB
Status Body Condition
202 { job_id } Job accepted
400 { error } Missing file, wrong MIME type, empty file
429 { error } Rate limit exceeded or server at capacity 
// Request
POST /api/documents/scan
Content-Type: multipart/form-data
Body: file=<image.jpg>

// Response 202
{ "job_id": "a3f2c1d4-..." }
GET /api/documents/jobs/<job_id>

Polls the status of a scan job. Returns processing, complete, or failed. Poll every 1.5 seconds; client enforces a 60-second timeout.

Status Body Condition
200 { status, result? } Job found
404 { error } Job ID not found 
// Response — complete
{
  "status": "complete",
  "result": {
    "text": "extracted text here...",
    "char_count": 342,
    "word_count": 61,
    "processing_time_ms": 1240,
    "image_b64": "iVBORw0KGgo...",
    "detection_count": 47,
    "doc_type": "printed",
    "doc_type_confidence": 0.7821
  }
}

// Response — failed
{
  "status": "failed",
  "error": "No four-point document contour found."
}

GET /api/documents/metrics

Returns aggregated statistics across all scans plus the 10 most recent records. Used to populate the Chart.js dashboard.

Status Body
